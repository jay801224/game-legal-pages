Privacy Policy — Threads Community Reply Assistant

This tool is a single-user, local command-line application operated solely by the owner of the Threads account @fattyjaysu. It is not offered to the public; there is no sign-up, no user accounts, and no third-party users.

What the Tool Does

The tool discovers public Threads posts related to the owner's content niche (AI workflows, automation, product building) via the official Threads API keyword search, presents those posts to the account owner as reply candidates, and — only after the owner personally reviews and approves a reply — publishes that reply through the official Threads API, capped at a fixed daily limit.

Data We Access and How It Is Used

Data Source Use Storage
Public post text, post ID, permalink, author username Threads API keyword search Shown to the owner as reply candidates; risk-classified by topic safety rules Local CSV working queue on the owner's machine only
The owner's own posts, replies, and insights Threads API Content scheduling and performance review for the owner's own account Local files on the owner's machine only
Access tokens Meta developer dashboard API authentication Local .env file on the owner's machine; never committed or transmitted elsewhere

We do not collect or process: passwords, private messages, non-public posts, contact lists, or any data about minors. Topic safety rules hard-block processing of posts containing personal data, politics, religion, protected-class topics, medical/legal/financial advice, and security-sensitive content.

Data Sharing and Selling

No data is sold, shared, transferred, or disclosed to any third party. All data stays on the owner's local machine.

Data Retention

Candidate data is kept only as long as needed for the working reply queue and audit log, then archived or deleted locally.

Data Deletion

To request deletion of any data this tool holds (for example, if your public post appears in the local candidate queue), email jay801224@gmail.com with the post URL. The corresponding local records will be deleted within 7 days. The tool holds no server-side data, so deletion of local records is complete deletion.
